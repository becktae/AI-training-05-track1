# input-policy.md — 사용자 입력 파싱·검증 규칙 (파이프라인 ①)

## 확정해야 할 3가지
요청 처리 전 반드시 아래 항목을 확정한다.

| 항목 | 설명 | 불명확 시 |
|------|------|----------|
| 대상 STEP | 1~7 중 하나 | 사용자에게 질문 |
| 변경 유형 | 추가 / 수정 / 제거 / 리디자인 | 문맥 추론 → 확인 |
| 변경 범위 | 해당 STEP 섹션 ID | 섹션 테이블 참조 |

## 명확한 입력 — 바로 진행
- "STEP 3 일정 아코디언 시간 정보 추가" → STEP 3, 추가, `#itinerary`
- "날씨 카드 자세히 보기 버튼 스타일 변경" → STEP 4, 수정, `#weather`
- "체크리스트 ESTA 팁 문구 수정" → STEP 6, 수정, `#checklist`

## 모호한 입력 — 반드시 질문
- "카드 개선해줘" → 어느 STEP의 카드인지 확인
- "인터랙션 추가" → 대상 섹션 확인
- "전체 페이지 업그레이드" → STEP별 분리 처리 제안

추측 기반 진행은 금지. 틀린 섹션을 수정하면 기존 동작을 손상시킬 수 있다.

## 거부해야 할 입력
| 요청 내용 | 거부 이유 |
|----------|----------|
| CDN 라이브러리 추가 | TRD 외부 의존성 금지 정책 |
| React / Vue 전환 | TRD 단일 파일 원칙 |
| 2000줄 크게 초과 예상 변경 | TRD 파일 크기 기준 |
| git push 포함 | safety-policy.md 절대 금지 항목 |
| ~/.claude 수정 | safety-policy.md 절대 금지 항목 |

거부 시 이유와 대안을 제시한다.
