# information-research-policy.md — 정보 수집 정책 (파이프라인 ②③④)

## 우선순위 원칙
> 사용자의 현재 STEP 요청이 PRD/TRD/TDD보다 우선한다.
> PRD/TRD/TDD는 기본 기준이며, 사용자가 명시적으로 다른 내용을 요청하면 그것을 따른다.
> 단, 절대 규칙(implementation-contract.md 참조)은 override 불가하다.

---

## ② PRD/TRD/TDD 독해 순서

### 1. PRD.md 읽기
확인 항목:
- 해당 STEP의 **목록 뷰에서 보여주는 정보** — 현재 섹션에 이미 구현됐는지 체크
- 해당 STEP의 **클릭 시 열리는 상세 정보** — 텍스트 내용 정확히 확인
- 사용자 요청이 PRD와 다른 경우 → 사용자 요청을 따른다

### 2. TRD.md 읽기
확인 항목:
- **4.1 STEP별 상세 정보 표시 방식** — 모달 / 아코디언 / 팝오버 / 행 확장 중 해당 STEP 방식
- **4.2 모달 구현 규칙** — ESC 닫기, 배경클릭 닫기, scroll lock, 닫기 버튼 필수
- **4.3 Accordion 구현 규칙** — max-height 전환, aria-expanded 토글
- **2. 외부 의존성 금지** — 절대 추가 불가 목록 재확인
- **1.2 파일 크기 기준** — 2000줄 이하 확인

### 3. TDD.md 읽기
확인 항목:
- 해당 STEP과 연관된 TC 번호와 `visible` 판정 기준
- P0 (최우선) TC는 반드시 통과해야 함
- P1 이하 TC는 가능한 범위에서 통과

## ③ index.html 섹션 분석

읽기 범위: 해당 섹션 ID(`#overview`, `#cities` 등)의 HTML + 관련 CSS + JS
분석 항목:
- 현재 구현된 인터랙션 목록
- PRD 클릭 상세 정보 중 누락된 항목
- TRD 인터랙션 방식과 실제 구현 방식 일치 여부
- aria 속성 누락 여부

## ④ Firecrawl 수집 (조건부)

### 사용 조건 — 다음 중 하나 이상 해당 시만 사용
- PRD.md에 정의된 가격·시간 데이터가 현행 기준과 다를 가능성이 있을 때
- 사용자가 명시적으로 "최신 정보 반영해줘"를 요청했을 때
- 텍스트 콘텐츠가 부족하여 섹션을 채울 수 없을 때

### 사용 제한
- API Key 값은 절대 출력하지 않는다
- 수집 대상: 공개된 여행 정보 페이지 (Amtrak 가격, 박물관 시간 등)
- 금지: 로그인 필요 사이트, 개인정보 포함 페이지
- 수집 결과는 반드시 구조화 요약으로 정제 (원시 HTML 덤프 금지)
- 수집한 데이터는 `research/` 디렉토리에 저장 가능

### 불필요한 경우 — 스킵
- PRD.md 텍스트가 충분한 경우
- 사용자가 직접 내용을 제공한 경우
